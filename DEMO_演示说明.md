## 模块一：系统登录与运营工作台

### 🎬 演示视频

> 📂 视频文件：[`demo/01_登录与工作台.webp`](demo/01_登录与工作台.webp)

### 功能说明

此模块展示 HR 在登出后，一次性成功登录系统并查看全局视角的流畅操作——包括招聘工作台和实时数据看板。

**工作台 (Workbench)** 是系统首页，汇聚了当前用户的核心待办事项和近期动态：

- 📌 **我的待办事项**：待处理的简历解析、待安排的面试、待审批的 Offer 等。
- 📰 **最近动态**：系统范围内的操作流水，如候选人入库、AI 评估完成、面试状态变更等。

**实时看板 (Dashboard)** 提供数据驱动的招聘运营分析：

- 📊 **招聘漏斗**：投递 → 筛选 → 面试 → Offer → 入职的转化率可视化。
- 📈 **渠道分析**：各来源渠道（模拟招聘平台A/B/C/D/E）的简历采集数量统计。
- 📋 **岗位进度**：各岗位的招聘进展总览表。

### 演示操作步骤

1. 访问登录页面，输入 `demo-admin` / `demo123456` 登录。
2. 系统自动跳转到**工作台**，讲解待办事项和动态列表。
3. 点击左侧导航 **实时看板**，展示漏斗图、渠道分析图表和岗位招聘进度表。

## 模块二：数据采集与简历解析（AI Agent 辅助）

### 🎬 演示视频

> 📂 视频文件：[`demo/02_数据采集与解析.webp`](demo/02_数据采集与解析.webp)

### 功能说明

此模块展示系统如何从外部招聘平台**一键采集**候选人数据，并通过 AI Agent 完成**简历结构化解析**与入库。

整个数据采集流程遵循 **4 步标准化流水线**：

```
① 上传/采集 → ② 提取与解析 → ③ 预览和查重 → ④ 确认入库
```

**模拟招聘平台 (MockHire)** 是系统内置的演示数据源，模拟了真实招聘平台的核心数据结构：
- 候选人简历（姓名、技能标签、项目经历、联系方式）
- 投递岗位 JD（岗位职责与任职要求）
- HR 与候选人的初步沟通聊天记录

**数据采集页面**是 HR 操作数据导入的中枢：
- 支持本地文件上传（txt/md/pdf/docx）
- 支持平台适配器一键采集
- 导入任务队列管理（状态跟踪、操作记录）

**AI 解析预览面板**展示解析结果的可读化摘要：
- 🏷️ 技能标签识别（Python、FastAPI、SQLAlchemy 等）
- 🎓 教育经历提取
- 📊 字段置信度评分
- 🔍 自动查重结果
- ⚠️ 缺失字段提示

### 演示操作步骤

1. 点击左侧 **模拟招聘平台**，展示 MockHire 人才市场的候选人信息。
2. 点击 **一键采集到 TalentFlow**，系统自动抓取简历、JD、聊天记录等多模态数据。
3. 点击左侧 **数据采集**，在导入任务队列中查看新生成的任务。
4. 点击 **开始解析**，AI Agent 在后台进行文本提取和语义理解。
5. 在右侧预览面板中审阅解析结果。
6. 确认无误后点击 **确认入库**，候选人正式进入人才库。

> 💡 **演示重点**：系统的 **Human-in-the-loop** 设计——AI 负责提取和建议，入库的最终决定权保留在 HR 手中。

## 模块三：AI 证据化评估与面试排期

### 🎬 演示视频

> 📂 视频文件：[`demo/03_AI评估与排期.webp`](demo/03_AI评估与排期.webp)

### 功能说明

此模块展示系统如何利用大模型对候选人进行**证据化评估**，以及自动化完成**面试邀约与排期**。

**AI 评估引擎**的核心特点是"不只给分数，更给证据"：

- 🎯 **综合评分**：基于岗位 JD 要求的多维匹配度打分（如 88.25 / 100）。
- 📝 **证据链**：每项评分都附带具体依据，例如：
  - "候选人在项目中使用 FastAPI 搭建问答服务接口 → 匹配后端开发要求 ✓"
  - "使用 SQLAlchemy 管理数据 → 匹配数据库技能要求 ✓"
- ⚖️ **可解释性**：评估结果可追溯、可审计，避免 AI"黑箱"决策。

**面试排期系统**实现了候选人自助确认的自动化流程：

- 📅 系统生成带 **HMAC 安全签名**的候选人确认链接。
- 🕐 候选人无需注册系统，通过链接即可查看可选时间段并确认面试。
- 🔄 面试状态自动流转：`draft → confirmed → completed`。
- 📝 面试官可在详情页填写反馈结论和下一步操作建议。

### 演示操作步骤

1. 进入 **AI 评估** 页面，查看评估任务列表。
2. 点击候选人查看评估报告：综合评分 + 逐项证据。
3. 进入 **面试排期** 页面，展示已排期的面试列表。
4. 点击某场面试查看详情：时间段、候选人确认链接、面试反馈表单。

> 💡 **演示重点**：排期链接的 HMAC 签名机制——候选人无需登录系统即可安全操作，兼顾安全性与用户体验。

## 模块四：面试反馈与 Offer 发放

### 🎬 演示视频

> 📂 视频文件：[`demo/04_面试反馈与Offer.webp`](demo/04_面试反馈与Offer.webp)

### 功能说明

此模块展示面试结束后的评价归档、Offer 审批全流程以及邮件发送。

**面试反馈归档**：
- 面试官填写结论（通过 / 待定 / 淘汰）和详细反馈。
- 选择下一步操作：发 Offer / 安排下一轮面试 / 进入人才池。
- 反馈提交后状态自动流转。

**Offer 管理**采用多 Tab 结构化设计：

| Tab 页 | 内容 |
|--------|------|
| **条款** | 候选人信息、岗位、入职主体、工作地点、薪资/补贴、职级、有效期 |
| **草稿** | AI 生成的邮件草稿和审批摘要 |
| **发放** | 一键发送 Offer 邮件操作 |
| **审计** | 完整的审批流程追溯记录 |

### 演示操作步骤

1. 在 **面试排期** 页面选择已完成的面试，展示反馈录入表单。
2. 进入 **Offer** 页面，展示 Offer 队列（候选人、岗位、状态、审批结果）。
3. 点击候选人查看 Offer 详情的 4 个 Tab。
4. 展示"发送 Offer"操作。

> ⚠️ **演示重点**：薪资、职级、入职日期等关键字段标注为"待 HR 填写"，**不会由 AI 自动决定**，确保人工主导关键决策。

## 模块五：入职跟进、通知同步与审计日志

### 🎬 演示视频

> 📂 视频文件：[`demo/05_入职跟进与审计.webp`](demo/05_入职跟进与审计.webp)

### 功能说明

此模块展示候选人入职后的全流程管理、系统通知机制以及企业级安全审计能力。

**入职跟进**：
- 状态机驱动的入职流程管理：背景调查 → 签署合同 → 准备工位 → 正式入职。
- 每个阶段有对应的 Checklist、负责人和截止日期。

**通知与同步**：
- 系统内部通知列表（面试确认、Offer 审批通过、状态变更等）。
- 外部集成：企业微信 Webhook 推送、SMTP 邮件同步。

**审计日志**（核心合规能力）：
- 📋 完整记录所有操作：时间戳、操作者（HR / AI Agent / 系统）、操作类型、对象、详情。
- 🔒 不可篡改的操作追溯：候选人数据变更、AI 评估触发、Offer 审批等。
- 🔎 支持按结果筛选和时间范围查询。

**系统设置**：
- AI 模型配置：运行模式切换（Mock ↔ 真实 API）、模型名称、API Base URL。
- 集成配置：企业微信、邮件 SMTP、腾讯会议、采集器适配器。
- 安全管理：API Key 存储与清空。

### 演示操作步骤

1. 点击 **入职跟进**，展示状态机流转和 Checklist。
2. 点击 **通知与同步**，展示通知列表和推送配置。
3. 点击 **审计日志**，展示全量操作追溯记录。
4. 点击 **系统设置**，展示模型配置和集成管理。

## 技术亮点总结

| 亮点 | 说明 |
|------|------|
| 🔒 **确定性工作流** | 核心业务节点由状态机控制，不依赖 AI 猜测，杜绝"幻觉"导致的业务错误 |
| 🤝 **Human-in-the-loop** | AI 负责提取、评估、建议；入库、审批、发放等关键决策保留人工确认 |
| 📝 **证据化评估** | AI 评分附带具体证据链，可追溯、可解释、可审计 |
| 🛡️ **隐私与安全** | 敏感数据本地存储，调用 LLM 前自动 PII 脱敏（电话、邮箱遮蔽） |
| 💰 **零成本部署** | SQLite + FastAPI，Mock 模式无需 API Key 即可完整闭环 |
| 🔐 **安全签名链接** | 面试排期使用 HMAC 签名，候选人无需注册即可安全操作 |
| 📊 **全链路审计** | 所有操作留下不可篡改的审计日志，满足企业合规要求 |
